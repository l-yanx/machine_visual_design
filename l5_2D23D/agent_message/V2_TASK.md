# V2_TASK.md

# Traditional Sculpture 3D Reconstruction V2

## Overall Goal

Build V2 of the traditional multi-view 3D reconstruction project based on the accepted V1 outputs.

V2 extends V1 from dense point cloud generation to:

```text
Dense point cloud
→ Mesh reconstruction
→ Model format export
→ Three.js frontend visualization
```

V2 contains two main tasks:

- **Task 3 / Subagent 3 V2:** Mesh Reconstruction
- **Task 4 / Subagent 4 V2:** Three.js Frontend Visualization

V2 should reuse the outputs generated by V1. V2 must not modify the accepted V1 outputs unless explicitly creating a copied or derived file under V2 output directories.

---


# Task 3: Mesh Reconstruction

## Responsible Agent

```text
Subagent 3 V2
```

## Task Goal

Convert the V1 dense point cloud into a usable mesh model and export it in formats suitable for local visualization and Three.js frontend loading.

The preferred first implementation is:

```text
dense.ply
→ point cloud cleaning
→ normal estimation
→ Poisson surface reconstruction
→ low-density mesh cleanup
→ mesh simplification
→ model export
```

Texture mapping is not required in V2. Vertex color preservation is preferred if supported by the export path.

---

## Task 3 Input

Required input:

```text
results/dense/dense.ply
```

Optional input:

```text
results/sparse/sparse.ply
results/sparse/cameras.json
data/images_resized/
```

---

## Task 3 Required Subtasks

### 1. Read Dense Point Cloud

Read the V1 dense point cloud.

Input:

```text
results/dense/dense.ply
```

Expected internal data:

```text
1. Point coordinates
2. Point colors, if available
3. Point cloud bounding box
4. Point count
```

---

### 2. Point Cloud Cleaning

Clean the dense point cloud before surface reconstruction.

Required operations:

```text
1. Remove NaN and infinite points.
2. Remove statistical outliers.
3. Optionally remove radius outliers.
4. Optionally crop extreme boundary noise if necessary.
5. Optionally downsample using voxel downsampling.
```

Recommended output:

```text
results/mesh/cleaned_dense.ply
```

---

### 3. Normal Estimation

Estimate and orient normals for the cleaned point cloud.

Required operations:

```text
1. Estimate normals using local neighborhood search.
2. Orient normals consistently if possible.
3. Save point cloud with normals if supported.
```

Recommended output:

```text
results/mesh/cleaned_dense_with_normals.ply
```

---

### 4. Poisson Surface Reconstruction

Generate an initial mesh using Poisson surface reconstruction.

Required operations:

```text
1. Run Poisson surface reconstruction.
2. Save the raw reconstructed mesh.
3. Save density values or equivalent quality information if available.
```

Recommended output:

```text
results/mesh/mesh_poisson_raw.ply
```

Recommended initial parameters:

```yaml
mesh:
  poisson_depth: 8
  poisson_scale: 1.1
  poisson_linear_fit: false
```

The exact parameters may be adjusted by Subagent 3 V2 based on point cloud quality.

---

### 5. Low-density Mesh Cleanup

Remove low-confidence or low-density mesh regions.

Required operations:

```text
1. Remove vertices with low Poisson density.
2. Remove unreferenced vertices.
3. Remove degenerate triangles.
4. Remove duplicated triangles.
5. Remove duplicated vertices.
6. Optionally keep only the largest connected component if many floating fragments exist.
```

Recommended output:

```text
results/mesh/mesh_cleaned.ply
```

---

### 6. Mesh Simplification

Simplify the mesh to make it suitable for frontend display.

Required operations:

```text
1. Record original vertex and triangle count.
2. Simplify mesh if triangle count is too high.
3. Record simplified vertex and triangle count.
4. Preserve overall sculpture shape as much as possible.
```

Recommended output:

```text
results/mesh/mesh_simplified.ply
```

Suggested triangle target for V2 frontend:

```text
50,000 to 200,000 triangles
```

If the mesh is already lightweight, simplification may be skipped, but this must be recorded.

---

### 7. Export Model Files

Export the mesh into formats usable by visualization tools and Three.js.

Required output:

```text
results/mesh/model.ply
results/mesh/model.obj
results/mesh/model.glb
```

If GLB export cannot be completed directly in Python, Subagent 3 V2 must provide either:

```text
1. A documented conversion command, or
2. An alternative Three.js-loadable OBJ/PLY path.
```

---

## Task 3 Acceptance Criteria

Task 3 is considered complete if all the following conditions are satisfied:

```text
1. results/dense/dense.ply is successfully read.
2. cleaned_dense.ply is generated and can be opened in Open3D, MeshLab, or CloudCompare.
3. Normals are estimated for the cleaned point cloud.
4. A Poisson mesh is generated.
5. Low-density or floating mesh artifacts are reduced.
6. model.ply is generated and can be opened.
7. model.obj is generated or a documented reason is provided if skipped.
8. model.glb is generated or a documented Three.js-compatible fallback is provided.
9. The mesh roughly preserves the main sculpture surface structure.
10. The mesh does not contain large-scale flying surfaces that dominate the reconstruction.
11. mesh_log.txt records point count, mesh vertex count, mesh triangle count, and key parameters.
12. subagent3_V2_record.md is completed.
```

# Task 4: Three.js Frontend Visualization

## Responsible Agent

```text
Subagent 4 V2
```

## Task Goal

Build a Three.js frontend to visualize the V2 mesh model and selected V1/V2 reconstruction outputs.

The frontend should support at least:

```text
1. Mesh model display
2. Dense point cloud display
3. Sparse point cloud display
4. Basic camera pose visualization
5. Display mode switching
6. Orbit controls
```

---

## Task 4 Input

Required input:

```text
results/mesh/model.glb
```

Fallback input if GLB is unavailable:

```text
results/mesh/model.obj
```

Point cloud inputs:

```text
results/dense/dense.ply
results/sparse/sparse.ply
```

Camera pose input:

```text
results/sparse/cameras.json
```

---

## Task 4 Required Subtasks

### 1. Create Frontend Project Structure

Create the frontend directory.

Required structure:

```text
frontend/
├── index.html
├── package.json
├── src/
│   ├── main.js
│   ├── loaders.js
│   ├── scene.js
│   ├── ui.js
│   └── camera_poses.js
└── assets/
    ├── model.glb
    ├── model.obj
    ├── dense.ply
    ├── sparse.ply
    └── cameras.json
```

A simpler structure is acceptable if it remains clear and runnable.

---

### 2. Copy Required Assets

Copy or reference V1/V2 outputs into the frontend asset directory.

Required assets:

```text
frontend/assets/model.glb
frontend/assets/dense.ply
frontend/assets/sparse.ply
frontend/assets/cameras.json
```

If GLB is unavailable, use:

```text
frontend/assets/model.obj
```

---

### 3. Implement Three.js Scene

Required scene elements:

```text
1. Perspective camera
2. OrbitControls
3. Ambient light
4. Directional light
5. Grid helper or axes helper
6. Renderer attached to the HTML page
```

---

### 4. Implement Model Loading

Support loading the reconstructed mesh.

Preferred loader:

```text
GLTFLoader for model.glb
```

Fallback loader:

```text
OBJLoader for model.obj
```

Optional loader:

```text
PLYLoader for model.ply
```

---

### 5. Implement Point Cloud Loading

Support loading:

```text
frontend/assets/dense.ply
frontend/assets/sparse.ply
```

Required behavior:

```text
1. Dense point cloud can be displayed.
2. Sparse point cloud can be displayed.
3. Point size can be adjusted in code or UI.
4. Vertex colors should be used if present.
```

---

### 6. Implement Camera Pose Visualization

Read:

```text
frontend/assets/cameras.json
```

Visualize camera poses using simple markers.

Required behavior:

```text
1. Show camera centers as small markers.
2. Optionally show viewing directions.
3. Optionally connect camera centers to form trajectory.
```

---

### 7. Implement Display Mode Switching

The frontend should allow switching between:

```text
1. Mesh
2. Dense point cloud
3. Sparse point cloud
4. Camera poses
```

Minimum acceptable implementation:

```text
HTML buttons or checkboxes controlling object visibility
```

---

### 8. Add Basic Information Panel

Display basic reconstruction information.

Recommended fields:

```text
1. V2 stage name
2. Mesh file name
3. Dense point cloud file name
4. Sparse point cloud file name
5. Number of registered cameras, if parsed from cameras.json
```
